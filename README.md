# (C# 코딩) 심플 사칙연산기

## 개요

- C# 프로그래밍 학습
- 1줄 소개: 사용자의 버튼 입력을 실시간으로 처리하여 사칙연산과 다양한 수식 제어 기능을 제공하는 Windows Forms 계산기 프로그램
- 사용한 플랫폼:
  - C#, .NET Windows Forms, Visual Studio, GitHub
- 사용한 컨트롤:
  - 텍스트 표시: TextBox (입력값 전용 txtInput, 결과 및 수식 출력용 txtResult)
  - 입력 버튼: Button (숫자 패드 0-9, 소수점 btnDot, 부호 전환 btnPlusMinus)
  - 연산 및 제어: Button (사칙연산 4종, 결과 확인 btnEqual, 초기화 3종)
- 사용한 기술과 구현한 기능:
  - 컨트롤 명명 규칙 준수: 모든 컨트롤은 button1과 같은 기본 이름 대신 btnNum1, btnAdd 등 의미 있는 이름으로 변경하여 소스 코드의 가독성과 유지보수성을 극대화함
  - 데이터 형식 변환 및 처리: 사용자로부터 입력받은 문자열(string) 데이터를 정수형(int) 또는 실수형(double)으로 변환하여 연산하고, 계산된 결과를 다시 문자열로 가공하여 화면에 출력함
  - 이벤트 중심 프로그래밍: 각 버튼의 클릭 이벤트를 개별 또는 공통 메서드에 연결하여 사용자 입력에 즉각적으로 반응하는 UI를 구현함

## 실행 화면 (과제1)
- 과제1 코드의 실행 스크린샷

![과제1 실행화면](img/step1.png)

- 과제 내용
  - 기본적인 계산기 레이아웃 설계 및 주요 컨트롤(TextBox, Button)의 적절한 배치
  - 숫자 버튼 클릭 시 입력 내용이 텍스트박스에 누적되어 나타나는 입력 시스템 구축
  - 입력받은 두 개의 피연산자를 활용한 기초적인 더하기(+) 기능 및 결과 출력 로직 완성
  - 연산 과정에서 데이터 형 변환(Parse 및 ToString)의 안정적 적용
- 구현 내용과 기능 설명
  - 숫자 버튼(0-9)을 클릭할 때마다 현재 텍스트박스의 문자열 뒤에 해당 숫자를 이어 붙이는 로직을 구현했습니다. 특히 처음에 0이 표시된 상태에서 숫자를 누르면 0이 지워지고 새 숫자가 시작되도록 처리하여 실제 계산기와 동일한 입력 방식을 제공합니다.
  - 더하기(+) 버튼 클릭 시, 현재 입력된 값을 첫 번째 피연산자 변수에 저장하고 연산자 상태를 기록한 뒤 입력창을 비워 다음 숫자를 받을 준비를 합니다. 이때 상단 레이블이나 별도 텍스트박스에 현재까지의 수식을 표시하여 사용자가 흐름을 놓치지 않게 했습니다.
  - 결과 보기(=) 버튼을 누르면 저장된 첫 번째 값과 현재 화면의 두 번째 값을 정수형으로 변환하여 합산합니다. 계산이 완료되면 결과값을 문자열로 변환하여 메인 출력창에 크게 표시하고, 계산이 성공적으로 끝났음을 사용자에게 알립니다.

## 실행 화면 (과제2)
- 과제2 코드의 실행 스크린샷

![과제2 실행화면](img/step2.png)

- 과제 내용
  - 덧셈 외에 뺄셈(-), 곱셈(*), 나눗셈(/) 버튼을 UI에 추가하고 각 기능을 엔진에 통합
  - 모든 사칙연산 버튼이 공통된 연산 로직을 공유하도록 코드 구조 설계 및 이벤트 연결
  - 연산자 버튼 클릭 시 현재 연산 상태를 갱신하고 이전 입력값을 안전하게 보관하는 시스템 구축
- 구현 내용과 기능 설명
  - 과제 1에서 구축한 덧셈 로직을 확장하여 사칙연산 4종 세트를 완성했습니다. 각 연산자 버튼은 하나의 공통 이벤트 핸들러를 공유하거나 유사한 구조의 메서드를 호출하며, 내부적으로 현재 선택된 연산자가 무엇인지 기억하는 변수를 활용해 계산 로직을 분기했습니다.
  - 나눗셈(/) 연산 시 정수 나눗셈의 특성을 고려하여 결과가 정확하게 나오도록 처리하였으며, 곱셈(*)과 뺄셈(-) 역시 연산자 버튼을 누르는 즉시 상단 수식창에 기호가 반영되도록 하여 사용자 피드백을 강화했습니다.
  - 연속적인 연산이 가능하도록 설계하여, 이퀄 버튼을 누르기 전이라도 다른 연산자를 누르면 중간 계산 결과가 도출되어 다음 연산의 피연산자로 자동 활용될 수 있는 기초 토대를 마련했습니다.

## 실행 화면 (과제3)
- 과제3 코드의 실행 스크린샷

![과제3 실행화면](img/step3.png)

- 과제 내용
  - 오입력 수정 및 초기화를 위한 전문 제어 버튼(C, CE, Del) 기능 구현
  - 전체 시스템 초기화(C)와 마지막 입력값 삭제(CE)의 논리적 차이 구현
  - 문자열 조작을 통한 단일 문자 삭제(Del/BackSpace) 기능 추가
- 구현 내용과 기능 설명
  - C (Clear) 버튼은 프로그램 내의 모든 피연산자 저장 변수, 연산자 상태값, 수식 표시 레이블, 메인 출력창을 모두 비우고 초기 상태인 0으로 되돌리는 완전 초기화 기능을 수행합니다.
  - CE (Clear Entry) 버튼은 전체 계산 흐름과 이전에 저장된 값은 유지한 채, 현재 사용자가 오입력하여 화면에 떠 있는 마지막 숫자만 통째로 지우는 기능을 구현했습니다. 이는 긴 계산 도중 마지막 숫자만 틀렸을 때 매우 유용합니다.
  - Del (Delete) 버튼은 문자열의 Remove 메서드를 활용하여 텍스트박스에 입력된 가장 오른쪽 숫자 하나만 제거하는 백스페이스 기능을 담당합니다. 만약 글자를 지워 문자열 길이가 0이 되면 자동으로 0을 다시 표시하여 화면이 비어 보이지 않도록 예외 처리를 꼼꼼히 적용했습니다.

## 실행 화면 (과제4)
- 과제4 코드의 실행 스크린샷

![과제4 실행화면](img/step4.png)

- 과제 내용
  - 윈도우 기본 계산기를 벤치마킹한 고도화된 편의 기능 및 특수 수식 추가
  - 수학적 다양성을 위한 제곱(x^2), 제곱근(sqrt), 역수(1/x), 퍼센트(%) 연산 도입
  - 사용자 편의를 위한 소수점(.) 처리, 부호 전환(+/-) 및 연속 계산 로직 최적화
- 구현 내용과 기능 설명
  - 특수 기능 버튼을 추가하여 현재 입력된 숫자에 대해 즉각적으로 제곱근을 구하거나 제곱을 하는 등 공학적 연산 기능을 보강했습니다. Math 클래스의 메서드를 활용하여 정확한 수학적 결과값을 도출하도록 설계했습니다.
  - 부호 전환(+/-) 버튼은 입력된 수치에 -1을 곱하여 양수와 음수를 자유롭게 오갈 수 있게 하며, 소수점(.) 버튼은 문자열 내에 점이 이미 존재하는지 체크하여 중복 찍힘을 방지하는 로직을 포함했습니다.
  - 실제 사용자가 느끼는 편리함을 극대화하기 위해 결과 버튼을 누른 후 바로 숫자를 누르면 새로운 계산이 시작되고, 연산자를 누르면 결과값에서 계산이 이어지는 지능형 입력 제어 시스템을 구현했습니다.
  - 0으로 나누기와 같은 수학적 오류 상황에 대해서는 프로그램이 강제 종료되지 않도록 예외 처리를 수행하고 사용자에게 적절한 안내를 제공하여 프로그램의 안정성을 확보했습니다.
