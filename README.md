# (C# 코딩) 심플 사칙연산기

## 개요
- C# 프로그래밍 학습
- 1줄 소개: 버튼 및 키보드 입력을 실시간으로 처리하여 사칙연산과 고도화된 수식 제어 기능을 제공하는 Windows Forms 계산기 프로그램
- 사용한 플랫폼:
	- C#, .NET Windows Forms, Visual Studio, GitHub
- 사용한 컨트롤:
  - 텍스트 표시: TextBox (수식 기록용 TextBox_Input, 최종 결과 출력용 TextBox_Output)
  - 입력 버튼: 숫자 패드(0-9), 소수점(Dot), 부호 전환(PM)
  - 연산 및 제어: 사칙연산 4종(+, -, ×, ÷), 결과 확인(Result), 초기화 3종(C, CE, del), 특수 연산(Square, Sqrt, Reciprocal, Percent)
- 사용한 기술과 구현한 기능:
  - 컨트롤 명명 규칙: 기본 컨트롤 이름을 btnAdd, TextBox_Input 등으로 변경하여 유지보수성 향상
  - 데이터 형식 처리: string 데이터를 double 형식으로 변환하여 정밀한 실수 연산을 수행하고 결과를 다시 string으로 가공하여 출력
  - 이벤트 중심 설계: 버튼 클릭 및 키보드 입력을 개별/공통 메서드에 연결하여 사용자 반응형 UI 구현
  - KeyPreview 활성화: 폼 전체에서 키보드 입력을 가로채 마우스 없이도 모든 기능을 사용할 수 있도록 구현

- 과제1 코드의 실행 스크린샷

![과제1 실행화면](img/step1.png)

- 과제 내용
  - 계산기 레이아웃 설계 및 기초 컨트롤 배치
  - 숫자 버튼 클릭 시 텍스트박스에 숫자가 누적되는 입력 시스템 구축
  - 입력받은 두 개의 피연산자를 활용한 기초적인 더하기(+) 기능 및 결과 출력 로직 완성

  - 사용자 인터페이스의 기본이 되는 숫자 입력 시스템을 구축하였습니다. 각 숫자 버튼 클릭 시 텍스트 박스의 기존 문자열 끝에 새로운 문자를 덧붙이는 방식을 채택하였으며, 입력값이 비어있거나 초기 상태일 때 발생할 수 있는 예외를 고려하여 첫 숫자가 자연스럽게 배치되도록 처리하였습니다.
  - 연산의 기초인 덧셈 로직을 위해 첫 번째 피연산자를 변수에 임시 저장하고, 연산자 클릭 시 입력창을 비워 두 번째 피연산자를 맞이할 준비를 하는 상태 제어 시스템을 구현하였습니다.
  - 결과 확인 시에는 string 데이터를 double 혹은 int로 안전하게 캐스팅하여 연산을 수행하고, 산출된 결과값을 다시 화면에 출력하기 위해 최적화된 형식의 문자열로 변환하는 데이터 흐름을 완성하였습니다.

- 과제 내용
  - 뺄셈(-), 곱셈(×), 나눗셈(÷) 기능을 추가하여 사칙연산 엔진 통합
  - 모든 연산 버튼이 공통 로직을 공유하도록 코드 구조 설계 및 이벤트 연결
  - 연산자 버튼 클릭 시 현재 연산 상태를 갱신하고 이전 입력값을 안전하게 보관하는 시스템 구축

- 구현 내용과 기능 설명
  - 단순 덧셈을 넘어 사칙연산 전반을 아우르는 연산 엔진을 구축하였습니다. 연산자 버튼을 클릭할 때마다 현재 선택된 연산의 종류를 열거형이나 문자열 변수에 기록하고, 최종 계산 시점에서 이를 참조하여 조건부 분기 처리를 수행하도록 로직을 고도화하였습니다.
  - 사용자에게 노출되는 기호는 가독성을 위해 ×와 ÷를 사용하지만, 프로그램 내부 계산 로직에서는 표준 연산자로 변환하여 처리하는 매핑 시스템을 적용하였습니다.
  - 연산자 버튼을 연속으로 클릭하거나 변경할 경우, 마지막으로 선택된 연산자가 수식에 반영되도록 처리하여 사용자의 입력 실수를 유연하게 받아낼 수 있는 인터페이스를 제공하였습니다.

## 실행 화면 (과제3)
- 과제3 코드의 실행 스크린샷

![과제3 실행화면](img/step3.png)

- 과제 내용
  - 수정 및 초기화를 위한 C, CE, del 버튼 기능 구현
  - 전체 시스템 초기화(C)와 마지막 입력값 삭제(CE)의 논리적 차이 구현
  - 문자열 조작을 통한 단일 문자 삭제(Backspace) 로직 적용

- 구현 내용과 기능 설명
  - 사용자의 입력 경험을 개선하기 위해 세분화된 삭제 기능을 구현하였습니다. C (Clear) 기능은 모든 수식과 메모리에 저장된 피연산자, 현재 진행 중인 연산 상태를 완전히 소거하여 프로그램 실행 초기 상태로 되돌리는 강력한 초기화 기능을 담당합니다.
  - CE (Clear Entry) 기능은 전체 수식 흐름은 보존하되 현재 사용자가 입력 중인 가장 마지막 피연산자 영역만 비워줌으로써, 긴 수식 입력 도중 발생한 부분적 실수를 효율적으로 수정할 수 있게 돕습니다.
  - del (Delete) 기능은 문자열의 Substring 메서드를 활용하여 가장 오른쪽 글자 하나만 지우는 기능을 구현하였습니다. 특히 본 프로그램의 특징인 연산자 앞뒤 공백 처리를 고려하여, 연산자 삭제 시에는 공백을 포함한 세 글자를 한꺼번에 제거하는 예외 처리를 적용하여 수식의 문법적 무결성을 유지하였습니다.

## 실행 화면 (과제4)
- 과제4 코드의 실행 스크린샷

![과제4 실행화면](img/step4.png)

- 과제 내용
  - 윈도우 기본 계산기를 벤치마킹한 고도화된 편의 기능 및 특수 수식 추가
  - 수학적 다양성을 위한 제곱(x^2), 제곱근(sqrt), 역수(1/x), 퍼센트(%) 연산 도입
  - 사용자 편의를 위한 소수점(.) 처리, 부호 전환(+/-) 및 연속 계산 로직 최적화

- 구현 내용과 기능 설명
  - 단순 계산기를 넘어 전문적인 연산 환경을 제공하기 위해 Math 클래스를 활용한 특수 연산 기능을 대폭 강화하였습니다. 현재 입력된 값에 즉각적으로 제곱을 하거나 제곱근을 구하는 단항 연산 로직을 추가하였으며, 이를 통해 복잡한 수식의 중간 과정에서도 빠른 변환이 가능하게 했습니다.
  - 소수점 입력 시 중복 입력을 원천 차단하는 로직을 적용하여 데이터 오염을 방지하였으며, 부호 전환 버튼으로 양수와 음수를 자유롭게 오갈 수 있도록 하였습니다. 또한 퍼센트 연산을 통해 비율 계산의 편의성을 더했습니다.
  - 지능형 연속 연산 시스템을 구축하여, 결과가 도출된 후 연산자를 누르면 결과값에서 계산이 이어지고, 숫자를 누르면 자동으로 새로운 계산이 시작되는 윈도우 계산기의 사용자 경험(UX)을 그대로 재현하였습니다.
  - DataTable.Compute 메서드를 전격 도입하여 연산자 우선순위와 괄호 연산이 포함된 복잡한 다중 혼합 수식도 별도의 알고리즘 구현 없이 정확하게 산출하도록 로직을 혁신하였습니다. 추가된 괄호 기능은 중첩된 연산 구조에서도 수학적 규칙을 완벽히 준수합니다.
  - 나눗셈 연산 시 분모가 0이 되는 상황을 사전에 감지하여 에러 메시지를 출력하는 예외 처리 시스템을 통해 프로그램의 견고함을 확보하였습니다.
  - 키보드 매핑을 통해 숫자키, 엔터, 백스페이스, 각종 연산 기호를 키보드로 즉시 입력할 수 있도록 구현하여 작업 속도를 극대화하였습니다.
